Eu penso o software em torno de três perguntas:
  [1] o que quebra primeiro?
  [2] como ele se recupera?
  [3] quem vai manter isso daqui a 12 meses?

Simplicidade é funcionalidade. Performance se mede, não se assume.
Documentação acompanha o código.
